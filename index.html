What is it?

  • Making sure your dynamic page components are usable with assistive technology (e.g. screen readers)
  • Managing user focus
    • tabindex
    • element.focus()
    • Use buttons

When should you use this?

When it makes sense to

  • If you have the budget
  • If you expect your users will need it

Accordion

  1. Put an aria-expanded attribute on the trigger.
  2. Put an aria-hidden attribute on the panel.
  3. Put an id on the panel and an aria-controls attribute on the panel's trigger. The aria-controls attribute should equal the id of the panel.
  4. When a panel is toggled, toggle aria-expanded and aria-hidden states.

Tabs

  1. Put a role="tablist" attribute on the tabs container.
  2. Put a role="tab" attribute on each tab trigger.
  3. Put a role="tabpanel" attribute on each tab panel.
  4. Put an id on each trigger and an aria-labelledby attribute on each panel. The aria-labelledby attribute should match the id of its trigger.
  5. Select the current tab by adding an aria-selected attribute to it.
  6. Hide inactive panels from the screen reader by adding a aria-hidden attribute to each.

Modal

  1. When the modal opens, focus on it
  2. When the modal closes, focus back on the previously focused element
  3. While the modal is open, keep focus contained inside modal
