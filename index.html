What is it?

  • Making sure your dynamic page components are usable with assistive technology (e.g. screen readers)
  • Managing user focus
    • tabindex
    • element.focus()
    • Use buttons

When should you use this?

When it makes sense to

  • If you have the time/budget
  • If you expect your users will need it

Accordion

  1. Put an aria-expanded attribute on the trigger.
  2. Put an aria-hidden attribute on the panel.
  3. Put an id on the panel and an aria-controls attribute on the panel's trigger. The aria-controls attribute should equal the id of the panel.
  4. When a panel is toggled, toggle aria-expanded and aria-hidden states.

Tabs

  1. Put a role="tablist" attribute on the tabs container.
  2. Put a role="tab" attribute on each tab trigger.
  3. Put a role="tabpanel" attribute on each tab panel.
  4. Put an id on each trigger and an aria-labelledby attribute on each panel. The aria-labelledby attribute should match the id of its trigger.
  5. Select the current tab by adding an aria-selected attribute to it.
  6. Hide inactive panels from the screen reader by adding an aria-hidden attribute to each.

Modal

  1. Put a role="dialog" attribute on the container element.
  2. Put an aria-labelledby attribute that matches the id of the modal's heading element.
  3. Optionally, if the dialog contains additional descriptive text besides the dialog title, this text can be associated with the dialog using the aria-describedby attribute
  4. When the modal opens, focus on its default focusable control (usually an "OK" button).
  5. While the modal is open, keep focus contained inside modal.
  6. When the modal closes, focus back on the previously focused element.

Carousel

  1. Put a role="listbox" attribute on the container element.
  2. Put a role="option" attribute and a unique id on each of the slide elements
  3. Put an aria-activedescendent attribute on the container element and set it equal to the id of the active slide.
  4. Keep the aria-activedescendent attribute up-to-date as the carousel's active slide changes.
  5. Optionally, if you have previous and next buttons, it's probably best practice to give them each aria-controls attributes and set them to the id of the container element.
